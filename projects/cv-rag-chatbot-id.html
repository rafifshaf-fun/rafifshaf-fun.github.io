Chatbot RAG CV

Chatbot Retrieval-Augmented Generation interaktif — tanyakan apa pun tentang latar belakang profesional saya, didukung oleh LangChain + FAISS.

LangChain LCEL FAISS FastEmbed (ONNX) Groq Llama 3.3 Streamlit RAGAS 2024

Masalah

Profil GitHub saya tidak menceritakan semuanya. Sebagian besar karya terbaik saya terikat NDA, dan CV statis tidak bisa menjawab pertanyaan lanjutan. Saya membutuhkan cara agar recruiter dan calon klien bisa menjelajahi latar belakang saya secara interaktif — mengajukan pertanyaan spesifik dan mendapatkan jawaban yang sesuai konteks.

Ini juga merupakan demonstrasi langsung dari arsitektur RAG — menunjukkan bahwa saya bisa membangun, mengevaluasi, dan men-deploy aplikasi LLM, bukan sekadar membicarakannya.

Mengapa Ini Sulit

  • Kualitas retrieval adalah segalanya. Jika retriever menarik potongan teks (chunks) yang tidak relevan, LLM akan berhalusinasi. Saya menggunakan RAGAS untuk mengukur faithfulness, answer relevancy, dan context precision — melakukan iterasi pada strategi chunking dan model embedding hingga mendapatkan metrik yang solid.
  • Strategi chunking sangat penting. Data CV sangat terstruktur (bagian, poin-poin). Chunking yang naif akan merusak koherensi semantik. Saya mengimplementasikan chunking berbasis bagian (section-aware) yang menghormati struktur dokumen.
  • Pengembangan berbasis evaluasi (Evaluation-driven development). Daripada menebak-nebak apakah chatbot ini bagus, saya membangun pipeline evaluasi RAGAS dengan pertanyaan pengujian sintetis. Setiap perubahan pada chunking, prompting, atau retrieval dapat diukur.
  • Deployment di Streamlit Cloud. Tier gratis, tanpa manajemen server. Namun tantangan cold start dan batas memori mengharuskan saya untuk mengoptimalkan ukuran indeks FAISS dan proses pemuatan model.

Tech Stack

  • LangChain LCEL — komposisi chain secara deklaratif: pemuatan dokumen, pemotongan (splitting), retrieval, dan manajemen prompt
  • FAISS — vector store untuk pencarian semantik atas dokumen CV dan proyek
  • FastEmbed (berbasis ONNX) — embedding lokal, tanpa dependensi API luar, dan cold start yang cepat
  • Groq + Llama 3.3 — LLM untuk pembuatan jawaban melalui API inferensi Groq yang sangat cepat
  • Streamlit — framework UI, di-deploy di Streamlit Community Cloud
  • RAGAS — framework evaluasi untuk mengukur faithfulness, answer relevancy, dan context precision

Apa yang Akan Saya Lakukan Berbeda

  • Menambahkan hybrid search (BM25 + semantik). Pencarian semantik murni terkadang melewatkan kecocokan kata kunci yang persis (exact match). Pendekatan hybrid akan meningkatkan recall untuk kueri skill/teknologi yang spesifik.
  • Mengimplementasikan query rewriting. Pertanyaan pengguna sering kali ambigu. Menulis ulang (rewriting) pertanyaan tersebut menjadi kueri yang lebih spesifik sebelum proses retrieval akan meningkatkan context precision.
  • Menambahkan kutipan sumber di UI. Menunjukkan dari bagian CV mana setiap jawaban berasal akan membangun kepercayaan. Pengguna seharusnya bisa mengklik sumber referensinya.
  • Mengeksplorasi reranking. Penggunaan cross-encoder reranker setelah retrieval awal bisa lebih meningkatkan context precision, terutama untuk kueri yang ambigu.

Pelajaran Utama

Membangun chatbot RAG itu mudah. Membangun yang benar-benar berguna itu sulit. Perbedaannya ada pada evaluasi — mengukur kualitas retrieval, mengulangi strategi chunking, dan bersikap jujur saat sistem tidak mengetahui sesuatu. RAGAS membuat hal ini dapat diukur, bukan subjektif.
