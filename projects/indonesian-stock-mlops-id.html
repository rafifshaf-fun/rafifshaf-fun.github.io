Pipeline MLOps Saham Indonesia

Pipeline ML end-to-end dengan experiment tracking, monitoring, dan retraining otomatis untuk prediksi saham Indonesia.

🔒 NDA · Demo Sederhana
MLflow Grafana Docker Scikit-learn Pandas Time Series Python 2023–2024

Masalah

Bursa Efek Indonesia (BEI) menghadirkan tantangan unik untuk model ML: volatilitas tinggi, efek mikrostruktur pasar, dan dampak kalender regulasi yang tidak diperhitungkan oleh model yang dilatih di pasar Barat. Klien membutuhkan pipeline prediksi yang menghasilkan sinyal BUY/SELL untuk 45 saham blue-chip IDXBLUE — bukan hanya akurat, tapi dapat dimonitor, di-retrain, dan diaudit.

Kebutuhan utama: pipeline harus berjalan tanpa intervensi manual — mulai dari penarikan data, pelatihan, evaluasi, hingga deployment dengan observabilitas penuh. Repo publik ini adalah demonstrasi arsitektur yang disederhanakan; versi production-nya digunakan untuk keputusan trading langsung di bawah ikatan NDA.

Mengapa Ini Sulit

  • Data deret waktu keuangan bersifat non-stasioner: Kondisi pasar selalu berubah. Model yang berhasil kuartal lalu bisa gagal kuartal ini. Pipeline membutuhkan deteksi drift dan pemicu retraining otomatis.
  • MLOps dengan budget terbatas: Tanpa AWS SageMaker atau Databricks. Semuanya berjalan di VPS standar dengan Docker Compose — MLflow, Grafana, dan infrastruktur serving model harus hidup berdampingan secara efisien.
  • Feature engineering dalam skala besar: Data pasar Indonesia membutuhkan fitur khusus — sinyal rotasi sektor, rasio aliran dana asing vs domestik, dan penyesuaian kalender (seperti efek Ramadhan pada volume perdagangan).
  • Dapat diaudit: Prediksi keuangan harus bisa dijelaskan. Setiap versi model, kumpulan fitur, dan prediksi harus bisa dilacak riwayatnya.

Tech Stack

  • Scikit-learn — framework ML utama untuk model prediksi pergerakan saham
  • MLflow — experiment tracking, model registry, dan manajemen deployment
  • Grafana — dashboard real-time untuk akurasi prediksi, data drift, distribusi fitur, dan kesehatan sistem
  • Docker + Docker Compose — pipeline berbasis container dengan environment yang reproduktif
  • Pandas — penarikan data, feature engineering, dan preprocessing
  • Deteksi drift kustom — uji statistik pada distribusi fitur yang memicu workflow retraining otomatis
  • FastAPI — serving prediksi yang ringan

Apa yang Akan Saya Lakukan Berbeda

  • Menambahkan feature store. Feast atau store sederhana berbasis Redis akan mencegah inkonsistensi fitur antara fase training dan serving.
  • Menggunakan DVC untuk data versioning. Melacak dataset mana yang menghasilkan model mana sangatlah penting. Menggunakan Git saja tidak cukup.
  • Menerapkan shadow deployment. Menjalankan model baru dalam mode shadow sebelum mempromosikannya ke live production akan mengurangi risiko kegagalan.
  • Mengeksplorasi gradient boosting. XGBoost atau LightGBM berpotensi menangkap pola non-linear lebih baik daripada model Scikit-learn — patut dicoba pada iterasi berikutnya.

Pelajaran Utama

MLOps bukan tentang tool yang mewah — ini tentang reproduktibilitas dan kepercayaan. Ketika prediksi Anda memengaruhi keputusan finansial, Anda harus tahu persis versi model mana yang membuat prediksi mana, dengan data apa, dan mengapa. Itulah standar yang harus dipenuhi.
