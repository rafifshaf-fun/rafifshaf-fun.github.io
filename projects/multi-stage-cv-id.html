Pipeline CV Multi-Tahap

Arsitektur detect-then-classify untuk pengenalan objek dengan akurasi tinggi — dioptimalkan untuk edge deployment.

🔒 NDA · Demo Sederhana
YOLO SSD PyTorch TensorFlow OpenCV ONNX Edge Inference 2023–2024

Masalah

Klien membutuhkan sistem pengenalan objek untuk inspeksi kualitas industri. Tantangannya: ratusan kategori produk spesifik (fine-grained) yang tidak bisa ditangani secara akurat oleh detektor tunggal. Produk terlihat mirip di tingkat kategori, tetapi memiliki perbedaan halus yang sangat penting untuk keputusan kualitas.

Selain itu, sistem harus berjalan di perangkat edge — latensi inferensi di bawah 100ms, tanpa dependensi cloud, dan model harus cukup kecil untuk muat di memori di bawah 200MB.

Mengapa Ini Sulit

  • Pola detect-then-classify mengurangi kompleksitas kombinatorial. Daripada menggunakan satu model untuk menangani N kategori, detektor hanya mengurus lokalisasi dan masing-masing classifier menangani subset yang lebih kecil. Ini membuat proses training lebih cepat dan akurasi lebih tinggi.
  • Batasan edge deployment sangat nyata. Ekspor ONNX, kuantisasi model (INT8), dan manajemen memori yang hati-hati sangatlah esensial. Seluruh pipeline termasuk pre/post-processing harus muat dalam batasan latensi yang ketat.
  • Ketidakseimbangan data (data imbalance) antar kategori. Beberapa jenis produk muncul 100x lebih sering daripada yang lain. Pembobotan loss kustom, augmentasi data, dan pembuatan data sintetis (synthetic data generation) sering kali diperlukan.
  • Strategi update model. Saat produk baru ditambahkan, Anda tentu tidak ingin melatih ulang semuanya. Arsitektur bertahap ini memungkinkan penambahan classifier halus baru tanpa menyentuh detektor atau tahap classifier lainnya.

Tech Stack

  • YOLOv8 + SSD — deteksi objek, dilatih pada dataset kustom dengan augmentasi spesifik domain
  • PyTorch + TensorFlow — pelatihan classifier kustom menggunakan backbone ResNet/EfficientNet di kedua framework
  • ONNX Runtime — optimasi inferensi untuk edge deployment
  • OpenCV — preprocessing gambar, ekstraksi crop, dan post-processing
  • Kuantisasi model — kuantisasi INT8 yang mengurangi ukuran model hingga 4x lipat dengan penurunan akurasi yang minimal
  • Sistem evaluasi kustom — metrik per tahap, confusion matrix, dan profiling latensi

Apa yang Akan Saya Lakukan Berbeda

  • Mengeksplorasi YOLO-NAS atau RT-DETR. Arsitektur yang lebih baru menawarkan keseimbangan akurasi-latensi yang lebih baik daripada YOLOv8 untuk edge deployment.
  • Menambahkan kelas penolakan (rejection class). Saat classifier merasa tidak yakin, arahkan prediksi ke peninjauan manusia daripada memaksakan hasil. Ini sangat penting dalam inspeksi kualitas.
  • Menerapkan active learning. Secara otomatis menandai prediksi dengan tingkat keyakinan rendah untuk dilabeli oleh manusia guna meningkatkan model seiring waktu.
  • Menggunakan Triton Inference Server. Untuk orkestrasi multi-model di edge, Triton akan menyederhanakan proses pemuatan model, pembuatan versi (versioning), dan batching.

Pelajaran Utama

Pola detect-then-classify sering diremehkan. Memecah masalah menjadi beberapa tahap memberi Anda akurasi yang lebih baik, perawatan yang lebih mudah, dan fleksibilitas untuk memperbarui satu tahap tanpa menyentuh tahap lainnya. Sekarang ini menjadi pendekatan bawaan (default) saya untuk berbagai masalah multi-class vision.
