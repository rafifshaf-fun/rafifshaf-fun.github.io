Sovereign Ledger

Sistem manajemen keuangan komprehensif yang dibangun untuk koperasi pekerja kesehatan di Garut, Indonesia — di-deploy on-premise dan digunakan aktif setiap hari.

🔒 NDA · KPRI Warga Kesehatan
Laravel 13 Livewire 4 Alpine.js PostgreSQL Tailwind CSS Docker Full-Stack Solo Build

Masalah

KPRI Warga Kesehatan Kabupaten Garut — sebuah koperasi pekerja kesehatan — membutuhkan sistem manajemen keuangan yang lengkap. Selama bertahun-tahun mereka menggunakan Microsoft Excel, dengan seluruh riwayat transaksi terkunci di dalam file legacy tersebut. Solusi software yang ada di pasaran terlalu mahal atau tidak bisa menangani logika bisnis khusus koperasi (simpanan wajib, dana sosial, dan berbagai jenis pinjaman).

Tantangan utamanya: membangun dan men-deploy sistem keuangan level production sebagai solo developer — dengan sistem akuntansi double-entry, role-based access untuk lima jenis pengguna, pemrosesan batch bulanan otomatis, pelaporan profesional, dan migrasi data penuh dari catatan Excel lama mereka.

Mengapa Ini Sulit

  • Akuntansi double-entry dari nol: Setiap transaksi harus seimbang (balance). Hierarki Chart of Accounts yang lengkap, jejak audit, dan penguncian periode (period locking). Membangun ini dengan benar sebagai solo developer — tanpa latar belakang akuntansi — membutuhkan pembelajaran mendalam di domain ini.
  • Migrasi data legacy: Seluruh riwayat transaksi koperasi tersimpan di file Excel selama bertahun-tahun. Mengimpor, membersihkan, dan merekonsiliasi data ini ke dalam skema PostgreSQL yang ternormalisasi adalah tantangan data engineering tersendiri.
  • Lima peran pengguna yang berbeda: Admin, teller, anggota, auditor, dan super-admin — masing-masing dengan izin, tampilan, dan alur kerja yang berbeda. Role-based access control harus sangat ketat untuk sistem keuangan yang menangani uang asli.
  • Deployment Docker on-premise: Sistem ini berjalan di server fisik milik koperasi — menggunakan PHP-FPM + Nginx + PostgreSQL di dalam container Docker, dengan penjadwalan berbasis Supervisor untuk batch job bulanan otomatis dan backup volume yang persisten.

Fitur Utama yang Diselesaikan

  • Full Chart of Accounts — struktur akun hierarkis dengan aturan ketat double-entry
  • Pemrosesan batch bulanan otomatis — potongan simpanan wajib, kontribusi dana sosial, dan pencatatan cicilan pinjaman yang berjalan sesuai jadwal
  • Manajemen multi-pinjaman — dengan penjadwalan cicilan otomatis, perhitungan bunga, dan pelacakan pembayaran
  • Role-based access control — lima peran pengguna dengan hak akses granular
  • Pelaporan profesional — laporan PDF dengan kop surat resmi, ekspor CSV, dan buku besar (ledger) cetak menggunakan Laravel DOMPDF + PhpSpreadsheet
  • Rekonsiliasi data legacy — impor penuh dan rekonsiliasi riwayat transaksi koperasi selama bertahun-tahun dari Excel

Apa yang Akan Saya Lakukan Berbeda

  • Menambahkan automated testing lebih awal. Software keuangan tanpa testing itu menakutkan. Saya menambahkan pengujian PHPUnit di tengah jalan, tetapi memilikinya sejak hari pertama — terutama untuk pengecekan balance double-entry dan kalkulasi pinjaman — akan sangat menghemat waktu debugging.
  • Memaksimalkan fitur bawaan Laravel. Awalnya saya kurang memanfaatkan antrean (queues), notifikasi, dan sistem event Laravel. Merefaktor pemrosesan batch menggunakan Laravel queues dengan Supervisor membuat sistem jauh lebih tangguh.
  • Merancang sistem role-permission yang lebih granular. Lima peran berfungsi baik untuk rilis awal, tetapi seiring berkembangnya koperasi, izin yang lebih spesifik akan mengurangi kebutuhan custom override untuk setiap pengguna.
  • Membangun pipeline impor data khusus. Migrasi Excel legacy dilakukan secara semi-manual. Pipeline ETL yang dapat diulang akan membuat proses rekonsiliasi lebih cepat dan dapat diaudit.

Pelajaran Utama

Membangun sistem keuangan production untuk institusi nyata — sebagai solo developer — mengajarkan Anda lebih banyak daripada tutorial mana pun. Akuntansi double-entry bukan sekadar fitur, itu adalah aturan ketat yang memengaruhi setiap keputusan desain arsitektur. Sistem ini di-deploy on-premise di koperasi dan digunakan setiap hari. Itu adalah level tanggung jawab yang sangat berbeda dibandingkan side-project biasa.
